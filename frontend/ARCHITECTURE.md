# ImpactIQ — Frontend Architecture

This document details the architectural layout, patterns, and design principles applied to the ImpactIQ frontend codebase.

---

## 🏛️ Directory Architecture & Principles

We strictly follow a feature-based, atomic component structure:

1. **Separation of Concerns**: UI components are presentational and do not house business logic, direct API fetch invocations, or raw state mutation functions.
2. **Business Logic Isolation**:
   - **Services (`/services`)**: Manage communication protocols, mapping incoming data schema transformations.
   - **Hooks (`/hooks`)**: Adapt services into reactive React states utilizing TanStack Query (`useQuery` / `useMutation`).
   - **Store (`/store`)**: Keeps transient client-side UI configurations (e.g. sidebar toggle status, session variables) isolated from UI tree branches.
3. **Atomic Primitives (`/components/ui`)**: Highly reusable components (such as buttons, cards, select boxes) using custom CSS classes and Radix bindings.

## ⚡ The Analysis Pipeline & System Responsibilities

ImpactIQ relies on a clear boundary between deterministic calculation and generative AI representation.

```
Deterministic Code Engines (Compute Facts)
 └── Git Diff Parse
      └── Dependency Analysis
           └── Blast Radius Visualizer
                └── Security Scanner
                     └── Risk Engine (0-100 Score)
                          │
                          ▼ (Pass JSON Fact Payload)
                  Generative AI Engine (Provide Explanation)
                   ├── Executive Summary
                   ├── Risk Explanation
                   ├── Deployment Advice
                   └── Remediation Steps
```

### Deterministic Engine Responsibilities (No AI Allowed)
The system calculates absolute data using static analysis:
- **Blast Radius**: Traverses code imports and routes to build exact impact paths.
- **Risk Score**: Calculated using linear weighting of changes (LOC, API visibility change, severity of findings).
- **Security Scanner**: Detects exposed secrets, dependency vulnerabilities, and API omissions using AST rule engines.

### Generative AI Responsibilities
AI takes the structured JSON facts generated by the engine and outputs human-digestible insights:
- **Executive Summary**: High-level status of the commit.
- **Engineering Explanation**: Highlights where the codebase changed and what it means for other subsystems.
- **Remediation Suggestions**: Steps to make changes safe.

## 🎨 Visual Design Standard

The design mimics the professional SaaS design systems used by companies like **Vercel, Linear, and Datadog**:
- **Grid Layout**: 12-column grid spacing.
- **Spacing Scale**: Strict 8px increment bounds.
- **Visuals**: Light content grid offset by a dark sidebar navigation drawer, combined with modern typography (Inter) and custom glassmorphism.
- **Contrast**: Indigo accent colors highlight active operations and CTA vectors.
